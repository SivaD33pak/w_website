# Next Agent Handoff

## Project Goal

Build a modern wedding website for Sushmi & Nijin using the stack from `ARCHITECTURE.md`:

- Frontend: Next.js, TypeScript, Tailwind CSS, Framer Motion, GSAP
- Backend: FastAPI
- Database: Excel workbook via `openpyxl`
- Deployment targets: Vercel for frontend, Render/Railway for backend

The original UI was generated by Banani AI as static HTML/CSS. The current
direction is to migrate that design into a maintainable monorepo.

#### Library helpers added (in `frontend/lib/`)

- **`wedding-data.ts`** — hero quote/verse, contact info, food preference options.
- **`animations.ts`** — reusable Framer Motion variants: `fadeInUp`,
  `staggerContainer`, `staggerItem`, `scaleIn`, `fadeIn`, `digitFlip`.
- **`particles.ts`** — deterministic seeded PRNG (`mulberry32`) for SSR-safe
  star particles. Server and client render identical positions.

#### Animation strategy

- **Framer Motion**: section reveals, staggered children, scale-in images,
  countdown digit animations, mobile menu transitions, RSVP form field staggers.
- **GSAP + ScrollTrigger**: hero parallax on 4 background layers, countdown
  section scroll opacity effect.

#### RSVP form — real controls

The fake placeholder divs from the Banani export were replaced with real,
controlled form fields:

- Invitation Code (text, required)
- Guest Name (text, required)
- Number of Adults (number, required, 0–10)
- Number of Children (number, optional, 0–10)
- Food Preference (select: Vegetarian / Non-Vegetarian / Vegan / Jain / No Preference)
- Personal Message (textarea)

Form uses `useState`, client-side validation, a simulated submit with loading/
success/error states, and toast feedback. **It is NOT yet wired to the backend.**

### Hydration bugs — FIXED

Three hydration mismatches were found during `npm run dev` and resolved:

1. **`Math.random()` star particles** (CountdownSection, BibleVerseSection) —
   different values server vs client. Fixed with deterministic seeded PRNG
   in `lib/particles.ts`.
2. **Countdown timer** — server rendered at build time, client at request
   time. Fixed with a `mounted` flag that renders `00` during SSR then
   switches to real values after mount, plus `suppressHydrationWarning`.
3. **Hero `fill` images with zero-height parent** — added explicit
   `style={{ inset: 0 }}` on all 4 hero layer containers so the mountain
   and church images fill the full viewport.

### Favicon 404 — FIXED

Added `frontend/public/favicon.svg` and linked it in `layout.tsx`.

## Important Notes

- The original `AI_HANDOFF.md` describes the older static-export cleanup
  before the monorepo scaffold was created.
- `ARCHITECTURE.md` is the project specification. Some box-drawing characters
  are mojibake, but the requirements are understandable.
- `frontend/main.html` + `frontend/styles.css` are generated artifacts kept
  as visual reference only — the React components in `frontend/components/`
  are the source of truth now.
- Remote image URLs still depend on Firebase/Google Storage. For production,
  download approved images into `frontend/public/images/` and update
  `frontend/lib/wedding-content.ts`.
- The RSVP form currently simulates the submit with a `setTimeout`. There is
  no `frontend/lib/api.ts` yet and no connection to the FastAPI backend.
