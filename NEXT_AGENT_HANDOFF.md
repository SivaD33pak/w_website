# Next Agent Handoff

## Project Goal

Build a modern wedding website for Sushmi & Nijin using the stack from `ARCHITECTURE.md`:

- Frontend: Next.js, TypeScript, Tailwind CSS, Framer Motion, GSAP
- Backend: FastAPI
- Database: Excel workbook via `openpyxl`
- Deployment targets: Vercel for frontend, Render/Railway for backend

The original UI was generated by Banani AI as static HTML/CSS. The current direction is to migrate that design into a maintainable monorepo.

## Important Notes

- The original `AI_HANDOFF.md` describes the older static-export cleanup before the monorepo scaffold was created.
- `ARCHITECTURE.md` is the project specification. Some box-drawing characters are mojibake, but the requirements are understandable.
- `frontend/main.html` is still a generated artifact and should not be the long-term source of truth.
- `frontend/styles.css` includes generated Tailwind-like CSS plus manually extracted shared classes.
- Remote image URLs still depend on Firebase/Google Storage. For production, download approved images into `frontend/public/images/` and update `frontend/lib/wedding-content.ts`.
- The active export still has generated markup issues and should be used as visual reference, not as clean source.
